# 人工审查清单

自动化脚本（verify.py）覆盖了可机械检查的规则，以下清单覆盖需要人工判断的规则。
建议在论文定稿前逐项审查。

---

## 一、结构完整性

- [ ] **引言**是否包含研究背景、目的、意义
- [ ] **引言**是否客观评介前人研究并指出研究空白
- [ ] **资料与方法**是否完整描述研究设计、对象与纳入排除标准
- [ ] **资料与方法**是否说明研究工具及信效度
- [ ] **资料与方法**是否说明数据收集程序
- [ ] **资料与方法**是否说明统计学分析方法
- [ ] **涉及人体研究**是否说明伦理审查与知情同意
- [ ] **结果**是否按观察指标顺序依次报告，不作主观解释
- [ ] **讨论**是否结合文献解释结果
- [ ] **讨论**是否分析理论与实践意义
- [ ] **讨论**是否指出研究局限性
- [ ] **讨论**是否提出未来研究方向
- [ ] **结论**是否提炼核心发现（非各章小结重复）

## 二、语言与行文

- [ ] 全文术语是否统一（同一概念是否始终使用同一术语）
- [ ] 专业术语首次出现时是否添加缩写定义
- [ ] 缩写在摘要中的使用是否合理（摘要中首次出现也应定义，或使用全称）
- [ ] 是否杜绝口语化表述
- [ ] 行文是否谦逊，无"首次证实""颠覆性"等未论证词汇
- [ ] 同级并列内容是否合并在同一段落内（非逐项换行）
- [ ] 括号内是否无解释性语句（仅保留必要缩写定义）
- [ ] 是否无回引（如"见本文2.3.1节"）

## 三、数据与统计

- [ ] 同一指标在摘要、正文、表格中的数值是否完全一致
- [ ] P值格式是否规范（大写P、精确报告、不写前导零）
- [ ] 均值与标准差小数位是否一致
- [ ] 推断统计量（t、F、χ²、r等）是否保留2位小数
- [ ] 百分率是否按规则保留小数位（分母<100报告整数）
- [ ] 效应量与置信区间格式是否规范
- [ ] 前导零使用是否正确

## 四、文献引用

- [ ] 所有[@cite_key]引用是否指向真实存在的文献（通过 `library_search` 验证）
- [ ] 是否无[待引证]标记残留
- [ ] 引用密度是否合理（一个观点默认1篇，多篇需必要）
- [ ] 是否采用"边检索、边补充"策略（非批量补引）

## 五、AI痕迹

- [ ] 标题中是否无冒号
- [ ] 是否无自我夸大式表述
- [ ] 是否无回引
- [ ] 整体行文风格是否自然，无典型AI生成痕迹

## 六、用户意见

- [ ] 文稿中所有【】标记是否已处理
- [ ] 无法完善的【】标记是否已在其中说明原因

---

## 审查流程建议

1. 先运行 `python .harness/verify.py manuscript_v5.md` 获取自动化检查结果
2. 根据脚本输出的错误和警告，逐一修复
3. 修复后重新运行脚本，确认错误清零
4. 按本清单逐项人工审查
5. 全部通过后定稿
